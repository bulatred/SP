Отчёт по курсу «Разработка веб-API на Spring Boot»

Что было изучено?

В рамках курса я разработал систему мониторинга бортовых датчиков автобусного парка — полноценное Spring Boot-приложение с аутентификацией, ролями, загрузкой файлов, обработкой данных и интеграцией с внешними сервисами.

Основные технологии и концепции, которые я освоил:

- **Spring Boot MVC** — архитектура приложения, контроллеры, сервисный слой, работа с HTTP-запросами.
- **Spring Security + JWT** — stateless аутентификация через JWT-токены, защита эндпоинтов через `@PreAuthorize("hasRole('ADMIN')")`.
- **Ролевая модель** — две роли: `USER` (только просмотр) и `ADMIN` (полный CRUD + импорт/экспорт).
- **Работа с файлами** — загрузка и удаление фото автобусов, сохранение в локальную папку `/uploads`, безопасная проверка MIME-типов.
- **Импорт и экспорт данных** — загрузка показаний датчиков из CSV-файла, экспорт сводного отчёта в формат XLSX с использованием Apache POI.
- **Интеграция с Telegram** — отправка уведомлений о входе в систему через Telegram Bot API.
- **Swagger/OpenAPI** — автоматическая документация API через springdoc-openapi.
- **Логгирование** — полное покрытие SLF4J: вход, выход, операции с автобусами, ошибки.
- **Git и GitHub** — ведение истории изменений, коммиты по функциональности, структурированный репозиторий.

## Применение в будущем

Этот проект — **реалистичный микросервис**, который можно использовать в транспортной логистике или промышленном мониторинге. Навыки, полученные при его разработке, напрямую применимы:

- **Backend-разработка** — Spring Boot — один из самых востребованных фреймворков в Java-мире.
- **Безопасность** — понимание JWT, ролевой модели и защиты API.
- **Интеграции** — опыт работы с внешними API (Telegraм).
- **Работа с данными** — умение импортировать/экспортировать данные в разных форматах — частая задача в аналитических системах.
